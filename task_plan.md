# 任务计划：三国霸业地图编辑器

## 项目目标

围绕《三国霸业》建立一条可维护、可验证、可回写的地图编辑链路：

1. 解析并稳定导出 `.m/.stg/.dor/.s/.x` 等核心关卡数据。
2. 提供面向编辑任务的地图编辑器，而不是只面向底层记录的调试页。
3. 在保留原始字段可控性的前提下，逐步提升据点、城门、资源与校验的语义化编辑能力。
4. 所有稳定结论都要落到脚本、文档与测试里，而不是停留在对话里。

## 已完成主线

- `.m` 16 字节 cell 记录、`Hello1.0` 文件头与渲染链路已稳定。
- `stage.ini`、`.stg` 已具备结构化导出与安全回写能力。
- `.s/.x` 已建立“上 128 行派生、下 32 行保留尾区”的保守写回策略。
- `.dor` 已统一收口到 `src/san_tools/analysis/analyze_dor.py`，能够稳定导出分组、城门坐标、据点坐标与方向字段。
- 编辑器已经支持资源绘制、原始字段修改、patch 导出、`.m/.s/.x` 导出，以及据点与城门高亮联动。
