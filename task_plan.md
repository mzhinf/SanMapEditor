# Stage.ini 崩溃差异排查计划

## 目标

对比 `data/game/stage.ini` 与 `outputs/stage.ini` 的编码、字节布局和 INI 结构差异，找出可能导致程序崩溃的高风险变化；不修改待分析文件。

## 阶段

- [完成] 1. 获取仓库状态、文件元数据、哈希及编码证据。
- [完成] 2. 比较字节级布局、插入块和原始区段一致性。
- [完成] 3. 验证 Big5 标题并复原武将、城池逻辑行边界。
- [完成] 4. 汇总崩溃原因优先级并给出验证建议。
- [完成] 5. 提交本次新增的中文排查记录。

## 约束与决定

- 两个 INI 文件只读，不执行自动修复。
- 结论区分“已证实差异”和“可能的崩溃原因”。
- 所有脚本如需落盘，必须包含中文有效注释。

## 已遇到的错误

| 错误 | 次数 | 处理 |
|---|---:|---|
| 全局 `python.exe` 无法启动 | 1 | 改用 Codex 工作区依赖提供的 Python。 |
| 查询不存在的规划文件返回退出码 1 | 1 | 已确认三个规划文件原先不存在，现新建。 |
| 使用内置补丁工具时遇到 Windows 沙箱包装失败 | 3 | 改用 UTF-8 Git 补丁完成记录更新。 |
| PowerShell 使用当前运行库不存在的 `Encoding.Latin1` | 1 | 改用逐字节统计换行与 NUL。 |
| 通用 `SequenceMatcher` 被大量零字节拖慢 | 1 | 主动终止，改按 224/76 物理步长和逻辑行起点定向比较。 |
