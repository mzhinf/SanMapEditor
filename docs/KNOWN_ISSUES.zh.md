# 已知问题

更新日期：2026-07-13

## 游戏素材分发权

源码仓库不会提交原始游戏文件。当前 Windows 发布包可在本机生成，但其中可能包含 `heads.dat`、`stage.ini`、关卡文件或由游戏素材生成的图集。相关内容的公开再分发权尚未核验，因此暂不上传 GitHub Release。

状态：已知问题，暂时搁置。源码开发与本机测试不受影响。

## STG 研究实现尚未收口

仓库中保留了 KSY 字段模型、块流 roundtrip 编解码器以及早期分析视图。它们面向不同研究阶段，部分命名与抽象尚未统一。

编辑器的正式字段定义和顺序解析只以 `src/san_tools/ksy/m.ksy`、`dor.ksy`、`stg.ksy` 及 `src/san_tools/map/editor_model.py` 为准。其他 STG 模块只用于研究、统计或兼容导出，不得反向改变正式数据模型。

状态：研究收口暂时搁置。后续需要逐字段对照 KSY、补齐跨样本 roundtrip，再决定合并或移除并行实现。

## 小地图与运行态兼容性

`.s/.x` 当前采用“上 128 行从 `.m` 派生，下 32 行保留原始尾区”的保守写回规则。该规则已覆盖现有样本，但尚未证明与游戏内部所有调色和缓存流程完全等价。

新增据点、武将或大范围地图编辑后，仍应在游戏目录副本中验证导出结果。
