# 三国霸业地图编辑器 2.0 使用指南

本指南面向第一次接触地图编辑器的编辑人员。不要跳步骤；看见什么就按对应步骤做。

## 先记住三件事

1. **不要直接在唯一一份游戏目录上实验。** 先完整复制游戏目录，再把导出文件放进副本测试。
2. **编辑器不会自动保存。** 刷新网页、关闭浏览器或关闭启动器之前，必须点击“导出”或“Patch”。
3. **不要只移动 EXE。** `SanMapEditor.exe` 和同级的 `editor-data` 文件夹必须一直放在一起。

## 一、启动编辑器

### 使用发布 ZIP

1. 找到 `SanMapEditor-stageXX-日期.zip`。
2. 右键 ZIP，选择“全部解压”。不要在压缩包预览窗口里直接双击 EXE。
3. 打开解压后的文件夹。正常情况下应看到：

```text
SanMapEditor.exe
editor-data/
使用说明.txt
编辑器使用指南.md
```

4. 双击 `SanMapEditor.exe`。
5. 等待“小地图编辑器正在运行”窗口出现。
6. 默认浏览器会自动打开编辑器。若没有打开，点击启动器窗口中的“打开编辑器”。
7. 编辑期间保持启动器小窗口开启。关闭它会停止本地服务。

启动器只监听本机 `127.0.0.1` 的随机端口，不会把编辑器开放到局域网。

### 浏览器已经打开，但页面打不开

依次检查：

1. 启动器小窗口是否还在。
2. `SanMapEditor.exe` 旁边是否有完整的 `editor-data` 文件夹。
3. 是否移动过 EXE。若移动过，把它放回解压目录。
4. 点击启动器中的“打开编辑器”重新打开页面。

不要直接双击 `editor.html`。编辑器需要通过启动器提供的本地 HTTP 服务读取 JSON、图片和二进制参考文件。

## 三、最简单的完整操作

第一次使用时，先完成下面这一遍：

1. 启动编辑器，等待地图出现。
2. 顶部“工具”选择“选择”。
3. 在中央地图上左键点击一个格子。
4. 确认画布左上角的“格子：X,Y”已经变化。
5. 左侧“当前图层”选择 `acwx`，也就是底层地表。
6. 在资源库点击一个资源。工具会自动切换为“笔刷”。
7. 在地图目标格子上左键一次。每点一次就会写入一次。
8. 顶部点击“撤销”，确认地图恢复。
9. 点击“重做”，确认修改重新出现。
10. 右侧打开“校验”，确认没有红色阻断错误。
11. 点击顶部“导出”，等待浏览器下载 `stageXX-export.zip`。

完成这一遍后，再开始正式编辑。

## 四、导入项目

### 什么情况下不需要导入

发布包已经自带打包时的关卡。只编辑该基准关卡时，启动后可以直接工作，不必先点“导入”。

### 继续编辑以前导出的项目

1. 先把以前导出的 ZIP 解压到一个单独文件夹。
2. 点击顶部“导入”。
3. 在文件选择窗口中一次选中项目文件。至少要有 `stageXX.m`。
4. 推荐同时选择：

```text
stageXX.m
stageXX.dor
stageXX.stg
stage.ini
History.txt
heads.dat
stage_ini.xlsx       # 有则选择
相关 JSON            # 需要恢复管理快照或迁移 Patch 时选择
```

5. 点击“打开”。
6. 查看顶部状态文字，确认显示“已加载 stageXX.m，配套文件：……”。
7. 依次打开右侧“势力”“据点”“武将”，确认项目对象已出现。

注意：

- `.m` 决定地图 Cell 数据，必须选择。
- `.stg` 决定势力、据点和已加入剧本的武将。
- `.dor` 决定据点城门。
- `stage.ini` 决定城池和武将母表。
- `History.txt` 决定未加入剧本武将及历史字段。
- `heads.dat` 在本地导入时作为导出参考；浏览器中的头像预览使用发布包生成的 `heads.png`。更换另一套 `heads.dat` 后要重新制作发布包，才能得到对应预览。
- `.s/.x` 的有效区由 `.m` 重新生成，尾区使用当前发布包内的参考数据。继续编辑其他版本时，应使用同一关卡和同一资源版本制作的发布包。

### 导入 JSON 时要分清用途

- 同时选择 `.m` 和导出的项目 JSON：JSON 优先作为管理界面快照恢复，不会把同一批城门和据点再新增一次。
- 只选择 Patch JSON：按跨版本增量修改处理，若原值不一致会报告冲突。
- Patch JSON 不是完整游戏文件，不能直接复制进游戏目录。

## 五、地图移动、缩放和选格

| 操作 | 方法 |
| --- | --- |
| 选择一个 Cell | 工具切到“选择”，左键单击地图 |
| 选择矩形区域 | 先选第一个 Cell，按住 `Shift` 再点对角 Cell |
| 增加或取消一个 Cell | 按住 `Ctrl` 再左键单击 |
| 移动当前格 | 按方向键 |
| 平移地图 | 按住鼠标右键拖动 |
| 缩放地图 | 鼠标滚轮 |
| 快速跳到远处 | 点击右下角小地图 |
| 定位当前格 | 点击顶部“定位” |
| 按坐标搜索 | 输入 X 和 Y，点击“搜索”或按 `Enter` |

选区使用淡橙色显示。画布左上角“格子：X,Y”表示当前锚点；区域粘贴会以这个锚点对齐。

## 六、编辑地图图层

| 图层 | 含义 | 常见用途 |
| --- | --- | --- |
| `acwx` | 底层地表 | 草地、土地、水面等基础地形 |
| `acwy` | 叠加层 | 道路、边缘、过渡和覆盖地形；`-1` 表示没有 |
| `acwz` | 物件层 | 树、建筑、城墙等物件；`-1` 表示没有 |
| `terrain_tag` | 地形标记 | 绿色中心方框 |
| `blocked` | 阻挡标记 | 红色外框 |
| `site_trigger` | 据点势力范围 | 半透明范围填充 |
| `site_area` | 据点核心区域 | Cell 内部高亮 |

### 使用笔刷

1. 在左侧选择图层。
2. 在资源库点击一个资源编号。点击后会自动切到“笔刷”。
3. 在地图上左键目标 Cell。
4. 每次左键只修改当前 Cell，不要误以为拖动会自动连续绘制。
5. 修改后立刻看底部“修改数量”。数量异常时马上撤销。
6. 要重新查看格子而不写数据，顶部工具切回“选择”。

### 数据叠加和真实编辑不是一回事

- 左侧“数据叠加显示”只是控制是否看见标记，不修改文件。
- 选择 `terrain_tag/blocked/site_trigger/site_area` 图层并用笔刷写值，才是真实修改。
- 数据叠加只在底层、叠加和物件资源视图上额外显示；切到数据图层时显示该图层自己的原始选点标记。

### 小地图颜色

- 修改 `acwx/acwy/acwz` 后，编辑器会自动推导 `minimap_color`。
- 如果自动颜色不正确，选择目标 Cell，打开右侧“Raw”，手工修改 `minimap_color`，范围是 `0..255`。
- 手工值优先于同一次操作中的自动值。
- 修改后小地图会立即刷新。

## 七、区域复制、剪切和合成对象

### 先选择区域

1. 工具切到“选择”。
2. 点击区域第一个角。
3. 按住 `Shift`，点击对角位置。
4. 左侧应显示“选区：N 格，范围……”。没有这个提示就不要继续。

### 两种复制模式

| 模式 | 会复制什么 | 适合什么情况 |
| --- | --- | --- |
| 全复制 | 所有选中 Cell 的完整记录，包括底层和手工颜色 | 把一整块地形原样搬走 |
| 非底层复制 | 只复制存在叠加、物件或数据标记的 Cell；不复制 `acwx` 和 `minimap_color` | 把建筑、道路或标记盖到另一块底图上 |

### 复制和粘贴

1. 选择复制模式。
2. 点击“复制区域”，或按 `Ctrl+C`。
3. 左键选中目标位置。这个 Cell 是粘贴锚点。
4. 点击“粘贴区域”，或按 `Ctrl+V`。
5. 检查新位置和底部修改数量。

### 剪切

1. 选择区域和复制模式。
2. 点击“剪切区域”，或按 `Ctrl+X`。
3. 源区域会清除被复制的非底层内容，只保留底层地表。
4. 选择目标位置后粘贴。
5. 剪切和清理属于同一次事务，一次“撤销”即可恢复源区域。

### 保存为合成对象

“存为对象”把当前选区保存到本次浏览器会话的对象列表，之后可重复粘贴。它不是永久素材库；刷新页面前仍应导出项目或 Patch。

## 八、势力管理

1. 打开右侧“势力”。
2. 上半区选择势力，下半区编辑当前势力。
3. 点击“新增势力”创建势力。
4. 当前势力下会列出“已加入据点”。
5. “定位”会切到该据点并移动地图。
6. “移出”只解除当前势力归属，不是删除据点。
7. “加入已有据点”把其他势力或未归属据点转给当前势力。
8. “删除势力”风险较高。删除前先确认其据点如何处理，并立刻查看“校验”。

据点数是自动计算值，不需要也不应该手填。

## 九、据点和城门管理

1. 打开右侧“据点”。
2. 据点按城池、军寨、山寨顺序分类。
3. 从列表选择据点。编辑区会显示对应来源页签，例如 `ini`、`stg`、`dor`。
4. `ini` 编辑城池母表信息，例如规模、人口、金钱和粮草。
5. `stg` 编辑剧本中的据点属性、归属和内容。
6. `dor` 查看、增加、编辑或删除城门。
7. 据点坐标变化时，城门的据点 X/Y 会自动同步，不允许单独把它们改成另一坐标。
8. 点击“定位”检查据点和城门是否落在正确位置。

### 新增据点

1. 先在地图上选中据点核心 Cell。
2. 再打开“据点”并点击新增按钮。
3. 新据点会优先使用尚未占用的 `stage.ini` 城池母表行；没有空闲行时才复制城池模板。先检查自动带出的名称、类型、规模、人口、金钱和粮草。
4. 按实际用途修改类型（城池、军寨或山寨）、名称和属性。地图坐标已经取新增前选中的 Cell。
5. 把据点加入正确势力。
6. 添加城门，并在地图上逐个定位检查。
7. 打开“校验”确认 `stage.ini/.stg/.dor` 关联没有阻断错误。

### 给据点加入武将

在据点详情中的武将区域选择“已有武将”并添加。不要用“武将管理 / 新增武将”代替这个步骤；新增武将只创建母表候选，不会自动加入当前据点。

## 十、武将管理

1. 打开右侧“武将”。
2. 左侧列表包含已加入剧本武将，以及来自 `stage.ini/History.txt` 的未加入候选。
3. 点击武将后，在当前详情区域滚动和编辑；列表位置不会自动跳回顶部。
4. `ini` 页修改武将母表属性和头像编号。
5. `stg` 页修改当前剧本属性、据点归属、兵种和兵力。
6. `history` 页修改或新增 History 记录。
7. “新增武将”只新增 `stage.ini` 母表候选，并补充 History 记录。初值为：等级 1、带兵数 50、统御力/武力/智力各 60、忠诚值 100，三项最大值分别为 50/60/60。
8. 要让新武将真正出现在游戏据点内，再到“据点”中执行“添加已有武将”。
9. 删除或停用前，确认该武将是否仍属于某个据点。

武将列表只把统御力大于 0 的记录当作武将显示。头像来自发布包打包时的 `heads.dat` 图集。

## 十一、Raw 精确编辑

Raw 是当前 Cell 的字段级编辑区。普通地图制作优先使用图层和资源库，只有以下情况再用 Raw：

- 已经知道准确字段值。
- 需要修正自动推导的 `minimap_color`。
- 需要检查某个 Cell 的完整数据。

操作方法：

1. 先选中一个 Cell。
2. 打开“Raw”。
3. 找到字段，输入数值。
4. 按 `Enter` 或点击别处提交。
5. 数值越界时编辑器会拒绝并在顶部显示范围。
6. 单个字段旁的恢复按钮只恢复该字段。
7. “恢复当前记录”恢复当前 Cell 的全部可编辑字段。

不要修改自己不理解的值。Raw 的错误值可能生成游戏无法使用的文件。

## 十二、撤销和恢复的区别

| 命令 | 作用 |
| --- | --- |
| 撤销 | 撤回最近一次编辑事务 |
| 重做 | 重新执行刚撤销的事务 |
| 恢复图层 | 只把当前 Cell 的当前图层恢复到导入值 |
| Raw / 恢复当前记录 | 把当前 Cell 的全部可编辑字段恢复到导入值 |
| 清空修改 | 把全部地图 Cell Patch 恢复到导入值 |
| Patch | 下载当前增量修改 JSON，不生成可直接放进游戏的完整文件 |

区域剪切、区域粘贴以及 xyz 与自动小地图颜色都按一个事务撤销。

## 十三、校验和导出

### 导出前

1. 打开右侧“校验”。
2. 红色错误必须处理，编辑器会阻止导出。
3. 黄色警告要读完。警告不一定阻止导出，但可能表示缺少 `.dor/.stg/stage.ini` 或 `.s/.x` 参考尾区。
4. 检查底部修改数量是否符合预期。
5. 建议先点击“Patch”保存一个增量备份。

### 导出

1. 点击顶部“导出”。
2. 等待浏览器下载 `stageXX-export.zip`。
3. 把 ZIP 解压到新的临时目录，不要直接覆盖游戏。
4. 正常归档可能包含：

```text
stageXX.m
stageXX.s
stageXX.x
stageXX.dor
stageXX.stg
stage.ini
History.txt
heads.dat
stage_ini.xlsx
*_patch.json
```

5. `stage_ini.xlsx` 和 `*_patch.json` 是检查、迁移和备份文件，不要复制进游戏目录。
6. 先备份游戏副本中的同名文件。
7. 将需要的 `.m/.s/.x/.dor/.stg`、`stage.ini`、`History.txt` 复制进游戏副本。
8. `heads.dat` 通常只是原样带出；没有更换头像资源时不必覆盖。
9. 启动游戏，在对应关卡检查地图、势力、据点、城门和武将。
10. 游戏崩溃时立刻恢复备份，并保留导出 ZIP 和 Patch 用于排查。

## 十四、快捷键总表

| 快捷键或操作 | 功能 | 注意 |
| --- | --- | --- |
| `Ctrl+C` | 复制选区 | 按当前复制模式；光标在输入框内时仍是普通文字复制 |
| `Ctrl+X` | 剪切选区 | 会真实清理源区域 |
| `Ctrl+V` | 粘贴选区 | 以当前 Cell 为锚点 |
| `Ctrl+Z` | 撤销 | 输入框编辑时可能由浏览器处理文字撤销 |
| `Ctrl+Y` | 重做 | 与顶部“重做”相同 |
| `Ctrl+Shift+Z` | 重做 | `Ctrl+Y` 的替代方式 |
| `←/→/↑/↓` | 移动当前 Cell | 输入框获得焦点时用于移动光标或数值，不移动 Cell |
| `Shift+左键` | 从当前 Cell 到点击 Cell 选择矩形 | 先要有一个当前 Cell |
| `Ctrl+左键` | 增加或取消单个 Cell | 适合不连续选区 |
| 鼠标右键拖动 | 平移地图 | 不修改数据 |
| 鼠标滚轮 | 缩放地图 | 缩放范围有限制 |
| 点击小地图 | 把主视图移动到该位置 | 不会改变当前 Cell |
| 坐标框中 `Enter` | 搜索坐标 | X、Y 必须在地图范围内 |

## 十五、常见问题

### 点击地图却改坏了格子

检查顶部“工具”是否是“笔刷”。立刻点击“撤销”，然后切回“选择”。

### 势力、据点或武将没有出现

重新导入并确认同时选择了同一项目的 `.m`、`.stg`、`stage.ini`；城门还需要 `.dor`，未加入武将还需要 `History.txt`。

### 武将头像不对

头像预览来自发布包中的 `heads.png`。如果项目使用另一份 `heads.dat`，需要用那份资源重新制作发布包。

### 导出按钮报错

打开“校验”，先处理红色错误。顶部状态文字太长时可以点击错误区域收起浮层，再继续操作。

### 关闭页面后修改没了

这是因为当前版本没有自动保存。以后在刷新或关闭前先导出 ZIP，至少也要下载 Patch。

### 导出的游戏崩溃

1. 不要继续覆盖原文件。
2. 恢复游戏副本备份。
3. 保留本次导出 ZIP、Patch 和导入源文件。
4. 记录最后一次新增或修改的势力、据点、城门、武将。
5. 用更少的修改重新导出，定位是哪一类数据造成问题。
