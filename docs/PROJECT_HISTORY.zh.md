## 2026-07-08：STG 块流模型修订

- 把 `.stg` 主结论从早期固定 76 字节视图修正为 Emperor.exe 风格的多级 `u32 size + payload` 对象流。
- 使用 42 个 `.stg` 样本验证对象边界和 byte-for-byte roundtrip。
- 确认 `entity_part1 +0x00..+0x20` 是固定零保留区，修正旧的所属势力解释。
- 确认 `site_part2 +0x27C..+0x28C` 是五个可选 Entity 控制位，`+0x2AC` 在当前样本中固定为零。
- 未完全命名的运行态字段和尾区继续原样保留；并行 STG 研究实现的全面收口暂时搁置。

## 2026-07-14：stage.ini 块流写入修订

- 对比游戏有效母表与崩溃输出，确认旧实现把兼容工作簿的 224/76 字节视图误作物理步长。
- 确认普通武将为 `size=224` 的 228 字节完整块，城池为 `size=92` 主块加 `size=0` 次块的 100 字节双块。
- 修正新增对象插入边界、标题类型槽、武将能力镜像和两级计数更新，并用真实游戏母表锁定回归边界。
- 进一步确认游戏以主块物理序号解析人物 ID；新增武将必须位于全部 277 个原主块之后，不能插在 ID 273..277 的特殊武将之前。

- 修复 `stitch-kingdom` 与 `export-m-layers` 的旧顶层导入，建立全部注册命令的帮助可用性测试。
- 新增完整命令执行参考，统一说明参数、默认产物、兼容工具边界和写回风险。
- 完整测试结果为 72 项通过，1 项因未准备 `stage11` 样本而跳过。

## 2026-07-14：完整项目导入快照修订

- 将配套二进制与导出 `scenario/dor/history` JSON 的同时导入定义为“项目快照恢复”：JSON 只恢复管理界面，避免把已写入 `.stg/.dor/History.txt` 的记录再应用一次。
- 保留单独 JSON 的跨版本 Patch 语义；配套二进制与 JSON 同时导入时，JSON 只恢复管理快照，不重复写入对象。

## 2026-07-14：本地 STG 二进制导入

- 本地 `.dor` 改为使用新解析的据点坐标建立城门归属；分项 JSON 在同时提供时仍只作为管理快照，不重复写入。

## 2026-07-14：本地 stage.ini 母表导入

- 主块数、城池数、主块结束和城池结束位置改为从导入文件重新计算，避免再次追加已有武将或城池。

## 2026-07-14：本地 History.txt 导入

- 完整项目导入不再要求配套 JSON；JSON 保留为可选管理快照和跨版本 Patch。

## 2026-07-14：编辑数据恢复与默认值完善

- 完整项目导入把配套 JSON 作为管理界面快照，避免将已写入二进制的城门、据点和历史记录重复应用；单独导入 JSON 时仍保持跨版本 Patch 语义。

- 固定新增武将与据点的业务初值，并确保武将加入据点时同步等级、兵力、属性、忠诚和三项最大值。

## 2026-07-16：无资源地图编辑器发布链路

- Windows 发布包改为严格的五文件无资源包；构建和正式运行时均不读取仓库的游戏或文本目录。
- 启动器提供空项目、导入中、导入失败和已加载四种状态，用户从合法本地副本导入完整 `stageNN` 资源后，在系统临时目录创建会话。
- 建立输入校验、SHA-256 记录、会话复用、失败回滚、关闭清理和过期回收；地图、图集、小地图与头像只在运行时会话中生成。
- `stage.ini` 写回改用内置字段定义与真实块流布局，CP950 映射完整验证；缺失或异常的 `.s/.x` 用户尾区将阻断导出。
- 新增发布目录、ZIP 和 PyInstaller TOC 审计；最终完整回归为 124 项通过、1 项按样本条件跳过。游戏内兼容性与调色板发布权利仍保留为人工门禁。
