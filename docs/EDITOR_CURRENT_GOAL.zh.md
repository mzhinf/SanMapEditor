# 地图编辑器 2.0 当前目标

## 目标范围

本轮完善编辑器对 `.m/.dor/.stg/stage.ini/History.txt` 的一致性管理，重点解决新增城池和新增武将无法安全写入 `stage.ini` 母表的问题。

## 功能目标

1. 按已解析格式重建 `stage.ini` 主表和尾表，新增城池、武将时同步创建母表记录，不再以“缺少合法母表行”为由阻断导出。
2. 为 `.m/.dor/.stg/stage.ini/History.txt` 提供统一的新增、查询、修改、删除数据接口，并在关联对象变化时同步派生字段。
3. Raw 字段编辑支持一键恢复当前记录，恢复后撤销对应字段修改并立即刷新地图和校验结果。
4. 增加地图坐标搜索，输入格子坐标后选中并定位到目标 Cell。
5. 修复导入本地 `.m` 后小地图无内容的问题；小地图必须由当前 `.m` 记录实时重建，不依赖旧关卡图片。
6. 更新数据模型转换文档，逐字段说明编辑器模型、来源文件、二进制位置、转换规则、写回规则和关联约束。

## 现有功能语义

- **定位**：把当前选中 Cell 移到画布可视区域中心，不修改任何文件数据。
- **恢复图层**：恢复当前选中 Cell 在当前编辑图层上的原始值，只撤销该 Cell 的对应字段修改。
- **校验**：只读检查当前内存模型和导出前置条件，展示错误与警告，不自动修改数据。

## 验收标准

- 新增城池和新武将后可以导出 `stage.ini`，重新解析后记录数、ID 和字段值与编辑器模型一致。
- 新增城池、武将同时可写入 `.stg`，城门可写入 `.dor`，武将登场信息可写入 `History.txt`。
- 删除或移动对象后，势力据点数、据点实体列表、城门据点坐标等派生数据即时更新。
- Raw 恢复、坐标搜索和本地 `.m` 小地图均有自动化运行时测试。
- `stage01` bundle 重建成功，浏览器无脚本错误，主要操作完成一次端到端验证。

## 约束

- `.m/.dor/.stg` 的字段和顺序以 `src/san_tools/ksy/` 为唯一结构依据。
- 未确认字段保持原始字节，不以其他资料补造语义。
- 所有二进制重建必须通过“解析、修改、重建、再次解析”测试。
