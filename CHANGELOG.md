# 变更记录

本项目遵循“未发布”优先的维护方式，首次稳定版本发布后再采用语义化版本编号。

## 未发布
- 修复无资源 Windows EXE 选择地图后找不到 `editor_app.html`：构建时显式收集运行时模板，并由 `--check` 验证冻结资产。
- 地图多选改为 Shift 拖动的 2.5D 等距菱形框选，支持添加、扣除、取消和实时边界预览；区域快照改用 `du/dv`，跨奇偶行粘贴保持轮廓。

- 正式 Windows 发布改为五文件无资源包，构建命令移除 `--stage`，不读取 `data/game` 或 `data/text`。
- 桌面启动器新增空项目、导入中、导入失败和已加载四种状态，可选择单个 `stageXX.m` 或完整资源目录。
- 新增运行时输入清单、场景编号匹配、大小与 SHA-256 记录，以及缺失、重复和损坏文件报告。
- 新增系统临时会话的事务创建、相同输入复用、失败保留旧会话、正常关闭清理和过期回收。
- 地图、资源图集、小地图和头像改为用户导入后生成，正式发布包不携带关卡或派生图片。
- `stage.ini` 运行时写回模型改用内置 49/17 字段和真实块流布局，不再依赖仓库文本母表。
- 新名称使用完整、逐字符验证的 CP950 映射；运行时禁止回退仓库游戏或文本目录。
- 删除浏览器 `.s/.x` 尾区零填充回退，缺少或长度异常的用户尾区会阻断导出。
- 新增发布目录、ZIP 和 PyInstaller TOC 资源审计，以及外部路径、大小和 SHA-256 Manifest。

- 新增武将默认使用等级 1、带兵 50、统御/武力/智力 60、忠诚 100，并同步三项最大值。
- 新增据点固定使用规模 1、人口 1200、金钱 3000、粮草 7500、开发/商业 40、治安 50，其余业务数值为 0。

- 修正 `stage.ini` 新武将与新城池写入：按真实 size 块流追加并分别维护主块数、城池数，移除错误的 224/76 步长补齐，并保证人物 ID 与主块物理序号一致。
- 完成地图编辑器 2.0 的地图、图层、区域、势力、据点、城门与武将管理流程。
- 增加 `.m/.dor/.stg/stage.ini/History.txt` 的解析、编辑和回写链路。
- 增加撤销重做、复制剪切、合成对象、小地图与头像图集支持。
- `minimap_color` 默认按当前地图的 `z→y→x` 优先模型自动派生，同时保留 Raw 手工修正与全复制能力。
- 新增随 Windows 发布 ZIP 交付的完整编辑器使用指南，并补齐发布链路文件清单。
- 统一发布包、桌面启动器和网页顶部的编辑器标题，网页从 `release-info.json.app_title` 读取名称。
- 统一命令入口为 `python -m san_tools`，移除旧 `tools` 转发目录。
- 统一本地资源目录为 `data/game` 与 `data/text`，移除本机绝对路径。
- 补齐 HTML 与 KSY 包数据、CI、维护文档和仓库元数据。
- 合并阶段性过程记录，新增文档索引和项目历史。
- 补齐统一命令入口迁移，新增完整命令执行参考与入口可用性测试。

- 修复完整项目导入的管理快照恢复：同时选择本地 `.m/.dor/.stg/History.txt` 与导出 JSON 时，场景、城门和 History JSON 只恢复 UI，不会再次应用新增记录，避免城门和据点重复。

- 增加本地 `.stg` 二进制解析：完整项目不依赖 JSON 也能恢复新增据点、已加入剧本的武将、字段偏移与对象流布局，并据此正确关联 `.dor` 城门。

- 增加本地 `stage.ini` 母表解析：完整项目导入后，新增武将可显示 ini 字段并使用本地字段偏移编辑；主块数、城池数及后续追加位置均从导入文件重新计算。

- 增加本地 `History.txt` 解析：完整项目无需 JSON 即可恢复新增武将的 History 记录、加入时间与候选武将关系，并保留 CP950 字符映射供后续导出。
